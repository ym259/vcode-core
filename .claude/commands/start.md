# /start - プロジェクト開始

新しいプロジェクトのオンボーディングフローを開始します。

## 手順

1. 日本語で挨拶する
   「こんにちは！一緒にWebアプリを作りましょう 🚀」

2. 何を作りたいか聞く。具体例を出す：
   - 学習管理システム（LMS）
   - ECサイト
   - ポートフォリオサイト
   - タスク管理アプリ
   - 予約管理システム
   - ブログ・メディアサイト

3. ターゲットユーザーを聞く
   「このアプリを使うのはどんな人ですか？」

4. 主要な機能を聞く（最初のバージョンでは3〜5個に絞る）
   「最初に作りたい機能を3〜5個教えてください」

5. 回答をもとに `PROJECT.md` を作成する：
   ```markdown
   # プロジェクト名

   ## 概要
   [ユーザーの回答をもとに]

   ## ターゲットユーザー
   [ユーザーの回答をもとに]

   ## 主要機能（v1）
   - [ ] 機能1
   - [ ] 機能2
   - [ ] 機能3

   ## 技術スタック
   - Next.js 15 (App Router)
   - Firebase (Auth, Firestore, Storage)
   - shadcn/ui + Tailwind CSS
   - Vercel (デプロイ)
   ```

6. Firebaseの設定状態を確認する
   - `.env.local` が存在しない、または Firebase 設定が空の場合 → `/firebase-setup` を案内
   - 設定済みの場合 → 次のステップへ

7. 最初のページの構築を開始する
   - まずサイドバーレイアウトを作成
   - 最初の機能のUIを構築
   - 動作確認を促す
