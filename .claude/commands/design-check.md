# /design-check - デザインチェック

現在のUIがデザインシステムのルールに従っているか確認します。

## 事前確認

チェック前に、`design-system.md` に「Active Style」セクションがあるか確認する。
- **ある場合**: そのスタイル固有のルール（shadow レベル、radius、hover パターン等）をチェック基準にする
- **ない場合**: デフォルトのチェック項目を使う

## チェック項目

以下の項目を `src/` ディレクトリ内のコンポーネントで確認してください。

**除外**: `src/app/gallery/` 以下はデザインプレビュー専用のため、チェック対象外とする。

### 1. コンポーネント使用
- [ ] shadcn/ui コンポーネントを使っているか（カスタムのボタンやカード等を自作していないか）
- [ ] Lucide React のアイコンのみ使用しているか

### 2. カラー
- [ ] ハードコードされた色（`text-[#xxx]`, `bg-[#xxx]`）がないか
- [ ] CSS変数（`text-foreground`, `bg-background`, etc.）を使っているか
- [ ] Active Style がある場合: そのスタイルで許可されたカラーパターンに従っているか

### 3. スペーシング
- [ ] 8pxグリッドに沿っているか（`p-2`, `p-4`, `p-6`, `p-8`）
- [ ] 奇数のスペーシング（`p-3`, `p-5`, `p-7`）を不必要に使っていないか
- [ ] Active Style がある場合: そのスタイルのパディング基準に従っているか（例: Corporate Sharp は p-3 推奨）

### 4. タイポグラフィ
- [ ] フォントウェイトが2種類以内か（`font-normal` + `font-semibold`）
- [ ] 見出しにセマンティックなHTML要素（`h1`-`h4`）を使っているか

### 5. レイアウト
- [ ] レスポンシブ対応されているか（`sm:`, `md:`, `lg:` ブレークポイント）
- [ ] コンテンツ幅に `max-w-5xl` か `max-w-6xl` を使っているか

### 6. シャドウ・ボーダー・角丸
Active Style に応じてチェック基準が変わる：

| 項目 | デフォルト基準 | Active Style がある場合 |
|-----|-------------|----------------------|
| Shadow | `shadow-sm` 以下 | スタイルの shadow ルールに従う |
| Radius | `rounded-lg` 統一 | スタイルの radius ルールに従う（rounded-sm〜rounded-2xl） |
| Border | `border-border` | スタイルの border ルールに従う |

### 7. 日本語テキスト
- [ ] すべてのUIラベルが日本語か
- [ ] 適切な行間（`leading-relaxed` 以上）が設定されているか

### 8. スタイル一貫性（Active Style がある場合のみ）
- [ ] ホバーエフェクトがスタイルのパターンに従っているか
- [ ] カード・パネルの見た目（背景色、影、ボーダー）が統一されているか
- [ ] サイドバーの幅・背景・ナビスタイルがギャラリーのプレビューと一致しているか

## 出力形式

問題がなければ：
「✅ デザインチェック完了 — 問題ありません」

問題がある場合は、ファイル名と行番号、修正提案を一覧で表示してください。
